# 演示账号与权限说明文档

> 此文档用于前端联调验证，包含了系统中不同层级的演示账号。
> 所有的演示账号统一密码均为：`Pass1234`

---

## 1. 演示账号列表

| 登录名 | 角色 | 所属租户 | 描述 |
| :--- | :--- | :--- | :--- |
| `superadmin` | 超级管理员 | 系统租户 | **最高权限**。`is_admin=true`。可以跨租户管理所有人员、部门和角色。 |
| `tenant_admin` | 租户管理员 | 演示租户 | **租户级最高权限**。负责“演示总公司”及其子部门的管理。 |
| `demo_user` | 普通员工 | 演示租户 | **基础权限**。属于“演示总公司 -> 技术中心”部门。 |

---

## 2. 组织架构 (演示租户)

演示租户 (`00000000-0000-0000-0000-000000000002`) 的预设部门如下：

*   **演示总公司** (Root)
    *   **技术中心** (Parent: 演示总公司)
    *   **运营部** (Parent: 演示总公司)

---

## 3. 预设角色权限

| 角色 Key | 名称 | 数据范围 (`data_scope`) | 包含权限点 (示例) |
| :--- | :--- | :--- | :--- |
| `admin` | 超级管理员 | `1` (全部数据) | `user:list`, `user:create`, `role:list`, `role:assign` 等 |
| `user` | 普通员工 | `3` (本人数据) | `user:list` (只读) |

---

## 4. 验证流程建议

### A. 验证超级管理员
1. 使用 `superadmin` / `Pass1234` 登录。
2. 调用 `GET /api/v1/users`，应能看到包括 `superadmin`, `tenant_admin`, `demo_user` 在内的所有人。
3. 调用 `GET /api/v1/depts/tree?tenant_id=00000000-0000-0000-0000-000000000002`，能看到演示租户的完整树。

### B. 验证租户隔离
1. 使用 `tenant_admin` / `Pass1234` 登录。
2. 调用 `GET /api/v1/users?tenant_id=00000000-0000-0000-0000-000000000002`，应只能看到该租户下的 `tenant_admin` 和 `demo_user`。
3. 尝试修改 `superadmin` 的资料，应被拒绝或由于查询不到而返回 404。

### C. 验证权限点 (Frontend)
1. 登录后通过 `GET /api/v1/users/me/permissions` 获取权限列表。
2. 检查 `demo_user` 是否**没有** `user:create` 权限。
3. 检查 `tenant_admin` 是否拥有完整的 `user:*` 权限。

---

## 5. 数据库重置与重新生成
如果数据被玩坏了，可以重新执行以下脚本恢复初始演示状态：
```bash
uv run python scripts/seed_demo_data.py
```
*(注：脚本具有幂等性，不会创建重复数据)*
